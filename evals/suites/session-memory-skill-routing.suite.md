# Session-memory skill routing and composition

The suite preserves deterministic invariants established during owner review:

- all twenty current skill packages have one capability identity;
- only the two admitted routers are advertised;
- positive, negative, paraphrase, collision, and branch cases select the
  expected smallest candidate set;
- short-description retrieval and full-contract reranking remain separately
  inspectable;
- compatible ABI chains produce a ready task-local DAG with stages and
  checkpoints;
- conflicting or version-incompatible selections block;
- portable skill packages contain no unbound host paths, symlinks, transient
  files, hidden network/install behavior, or undeclared mutations;
- mention, visibility, selection, loading, procedure observation,
  verification, completion, and consequence remain distinct.

The suite does not establish behavioral lift over no-skill, cross-runtime
identity, or central promotion readiness.
