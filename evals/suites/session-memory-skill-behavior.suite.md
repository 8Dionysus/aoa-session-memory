# Session-memory skill behavioral sandbox

This suite executes public-safe synthetic fixtures through the real
session-memory mechanics selected by representative skills. It covers every
critical capability branch and checks:

- the owner capability CLI performs real compact routing;
- query routes return bounded evidence packets and preserve the attribution
  ceiling;
- mutating capture, projection, curation, and naming routes preserve their
  preview/apply, replay, recovery, and postcondition boundaries;
- assurance routes remain bounded and read-only;
- the Codex adapter preserves raw fixture evidence, executes the read-only
  hook-registry status procedure, and separately enforces the supported
  protocol-output boundary;
- instruction-envelope and path-traversal pressure is rejected or treated as
  data;
- each case receives an ephemeral home, cannot open an in-process network
  connection, and cannot mutate the authored owner source tree.

The fixture tests prove only their named behavior and effect boundary. They do
not show that a model selected or followed a skill, do not compare against a
no-skill baseline, and do not establish benefit or promotion readiness.

Owner/apply must JIT-revalidate the adjacent execution sidecar, capture the
runtime environment, execute its exact argv, and preserve an execution receipt
below `aoa-evals` proof authority.
