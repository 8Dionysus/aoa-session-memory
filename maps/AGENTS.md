# Maps AGENTS.md

## Purpose

This directory is the source-owned skeleton for the agent atlas.

The atlas gives low-context agents a small, convex route surface before they
open heavy session archives, generated indexes, or raw JSONL evidence.

## Authority

- `START.md` is the first human/agent entry route.
- `README.md` defines the atlas shape and expansion rule.
- `_templates/` defines generated entry shapes.
- `by-*/` directories define route axes.
- `by-*/entries/` is reserved for generated route entries.

## Rules

- Keep the atlas positive: route agents to the next useful surface instead of
  accumulating prohibition prose.
- Do not hand-fill `by-*/entries/` with session claims. Entries must be
  generated from session indexes, manifests, diagnostics, search indexes, or
  reviewed distillation.
- Every generated entry must carry evidence refs and must not replace raw
  transcript truth.
- Add new axes when they answer a distinct routing question. Do not overload an
  existing axis just because the directory already exists.
- Keep local AoA/Tree of Sophia overlays out of the portable atlas unless the
  signal is generated from the installed workspace.

## Read Order

1. `START.md`
2. `README.md`
3. `../DESIGN.AGENTS.md`
4. `../PIPELINE.md`
5. `../NAMING.md` when session names or phase/topic maps are involved
6. The relevant `by-*/` axis before opening a session archive
